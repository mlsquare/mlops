Course Title:

MLOps: Principles and Practices of ML System Design, Development and Deployment[1]

Broad Course Objectives:

  1. Understand ML model development lifecycle
  2. Learn and Apply ML System Design Patterns
  3. Build observable, reproducible, reliable, performant ML systems
  4. Learn and apply Full Stack ML Infrastructure to boost productivity
  5. Understand and apply Trustworthy, Responsible AI frameworks

Some specifics:

  1. Appreciate the similarities and differences between developing systems and products with and without ML: software 1.0 vs software 2.0
  2. Develop a mental model and an abstract representation of an ML pipeline and the associated data flows
  3. Adapt and apply Software Engineering Best Practices to ML model development such as code readability, documentation, modularity, testing.
  1. Apply tools to  develop and deploy production grade ML systems covering its lifecycle
  2. Study and apply ML System Design Patterns
  3. Develop an understanding of the R4 framework: Recollect, Replay, Recall, Replace a prediction by a model, and operationalise them
  4. Familiarize with cloud platforms
  5. Analyze and interpret metrics to take decisions w.r.t training & deployment  

Prerequisites

  1. Exposure and skill in data handling, building models in Python, PyTorch
  2. Exposure and skill in developing code using Python, Git, IDEs like VS Code
  3. A foundation course in Machine Learning, Deep Learning, Data Modeling, working with (Big) Data

Schedule Outline:

Weeks 1-5:

  • Essentials: Basic Principles and MLOps with OSS
  • Two Assignments
  • By the end of the first five weeks, students should be able deploy models with logging, documentation, unit tests,  and APIs

Weeks 6-10:

  • Holistic understanding of ML development, beyond chasing typical performance metrics
  • one Assignment, one mini project and a midterm
  • By the end of the first ten weeks, students should be able to deploy models, observe their performance, make improvements, redeploy them.  Ensure that the ML pipeline is reproducible. And also, incorporate principles from Responsible AI and build ML systems which can consist of many models and tools.

Weeks 11-14:

Practice and Capstone Project and Presentations

Participation and involvement:

  • This is a hands-on course.
  • Individual participation: scribe the lecture notes (at least two of them), work on assignments, and a midterm mini project.
  • Team participation: work on a project as a team of not more than 3 members.

Class size

Expect a class of size no more than 25. The class should have met the prerequisites (grade more than 80% in all prereq courses)

Grading:

10%: Scribe lecture notes

30%: Three assignments, each 10%

15%: Midterm mini project

20%: In-class midterm MCQs, FIBs, Data Interpretation

25%: Capstone project
